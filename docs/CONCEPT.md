# Система прогнозирования спроса для розничного магазина

## 1. Краткое описание проекта

Нужно разработать веб-приложение для владельца розничного магазина, которое будет показывать статистику по товарам и прогнозировать продажи.

Система должна помогать отвечать на вопросы:

- сколько товара будет продано завтра;
- сколько товара будет продано через 7, 14 и 30 дней;
- сколько товара нужно заказать у поставщика;
- какую цену можно рекомендовать для товара.

В основе системы лежат исторические данные о движении товаров за 4 года. :contentReference[oaicite:0]{index=0}

---

## 2. Основная цель

Сделать рабочее приложение, в котором пользователь может:

- войти в систему;
- загрузить данные;
- посмотреть общую статистику магазина;
- открыть карточку товара;
- увидеть прогноз продаж;
- сформировать заявку поставщику;
- выгрузить результат.

---

## 3. Пользователь системы

На текущем этапе в системе один тип пользователя:

- **владелец магазина**

Роли пока не разделяются.

---

## 4. Источники данных

Данные в систему могут попадать двумя способами:

### Вариант 1. Через GraphQL
Система получает данные из внешней системы управления ассортиментом.

### Вариант 2. Через загрузку базы или файла
Пользователь вручную загружает данные в систему.

---

## 5. Какие данные используются

Для каждого товара могут использоваться такие данные:

- наименование;
- категория;
- поставщик;
- закупочная цена;
- цена продажи;
- дата операции;
- тип операции:
  - приход;
  - продажа;
  - списание;
  - инвентаризация;
- остаток.

Дополнительно для прогноза могут использоваться:

- праздники;
- погода;
- день недели;
- месяц;
- сезонность.

1. Пользователь входит в систему

## 10. Прогнозирование

Система прогнозирует продажи на:
- 1 день  
- 7 дней  
- 14 дней  
- 30 дней  

Используются:
- история продаж  
- лаги  
- календарные признаки  
- погода  
- праздники  

Результат: количество продаж товара.

---

## 11. Логика заявки

Если: прогноз > остаток
то: заказ = прогноз - остаток

---

## 12. Рекомендация цены

Рассчитывается на основе:
- текущей цены  
- закупочной цены  
- прогноза  
- категории  

Не сохраняется в БД.
